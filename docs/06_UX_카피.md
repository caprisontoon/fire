# [6차 산출물] UX·카피라이팅 — 화면 문구 전체 모음

> 작성 역할: UX·카피라이팅 에이전트
> 상태: **적용 완료** — 아래 문구는 `frontend/index.html`(안내·면책)과 `frontend/app.js`(결과 문장),
> `backend/main.py`(disclaimers)에 이미 반영되어 있다. 문구 수정 시 이 문서와 해당 파일을 함께 고친다.

---

## 1) 톤&매너 원칙

1. **불안을 키우지 않는다.** "탈락", "폭탄" 같은 단어 대신 "대상입니다 / 여유가 있습니다"처럼 상태를 담백하게 알려준다.
2. **결정 지원 도구의 말투.** 단정 대신 안내: "~로 추정됩니다", "~를 확인해 주세요".
3. **숫자에는 항상 맥락.** 금액만 던지지 않고 기준선과의 거리(여유/초과)를 함께 보여준다.
4. **전문용어는 첫 등장 시 풀어쓴다.** 예: "피부양자 (가족의 직장보험에 얹혀 있음)".

## 2) 고정 면책 문구

**상단 배너 (모든 화면):**
> ⚠️ 본 결과는 **단순화된 모델에 따른 참고용 추정치**입니다. 실제 세액·보험료와 다를 수 있으며, 실제 신고·산정은 세무사, 국세청(홈택스), 국민건강보험공단의 안내를 따르십시오. 입력하신 정보는 어디에도 저장되지 않습니다.

**하단 푸터:**
> 이 도구는 의사결정을 돕는 참고용 시뮬레이터입니다. 세무 신고·건보료 산정의 근거 자료로 사용할 수 없습니다.

**API 응답 disclaimers (3종):** 위와 동일 취지 + "2025년 제도 기준" 명시.

## 3) 입력 필드 안내 문구 (요지)

| 필드 | 핵심 문구 포인트 |
|------|------------------|
| 은퇴 나이 | "이미 은퇴하셨다면 현재 나이와 같거나 작은 값" |
| 연간 생활비 | "자산이 몇 살까지 버티는지 계산에 사용. 0이면 생략" |
| 금융소득 공통 | **"연금저축·IRP·ISA 계좌 안의 금액은 모두 제외"** (스텝 상단 고정) |
| 국내 주식형 ETF | "매매차익은 비과세라서 분배금만 입력" |
| 기타 국내 ETF | "분배금과 매도 차익을 합쳐서 입력 — 전부 금융소득에 합산" (예시 종목 포함) |
| 해외 양도차익 | "금융소득 2,000만 판정에 포함되지 않고 양도세(연 250만 공제 후 22%) 별도" |
| 공적연금 | **"건보료에는 50%만 반영되지만 피부양자 판정에는 100% 반영"** — 가장 중요한 경고 |
| 사적연금 | "연 1,500만 이하면 3.3~5.5% 저율 과세로 끝나고 건보료에도 안 잡힘" |
| 재산 과표 | "모르면 공시가격의 약 60%" |
| 사업·임대소득 | "피부양자 자격에 불리합니다" 경고 |

## 4) 결과 화면 동적 문구 (app.js)

| 상황 | 문구 |
|------|------|
| 종합과세 대상 | "금융소득 N원 — 기준선(2,000만 원)을 N원 **초과**했습니다." (빨간 카드) |
| 종합과세 비대상 | "금융소득 N원 — 기준선까지 N원 **여유**가 있습니다." (초록 카드) |
| 건보료 카드 | "적용 자격: 직장/지역/피부양자 (장기요양 포함)" + 피부양자 불가 사유 나열 |
| 결과 상단 | "아래 판정·금액은 **현재 나이 기준 1년치**입니다. 나이에 따른 변화는 그래프와 표에서 확인하세요." |
| 오류 (422) | "입력값에 문제가 있습니다. 나이 범위(19~89세)와 음수 여부를 확인해 주세요." |
| 오류 (네트워크) | "계산 요청에 실패했습니다. 서버가 켜져 있는지 확인해 주세요." |

## 5) 단순화 고지 (결과 화면 접이식)

- 세부 공제 생략으로 세금이 실제보다 **약간 많게(보수적으로)** 추정되는 경향 명시
- 지역 건보 재산분 ±20% 오차 가능성 명시
- 2025년 제도 기준 고정·물가 미반영 명시
