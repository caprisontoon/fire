# [2차 산출물] 세무·건강보험 전문가 — 계산 규칙 상세 설계

> 작성 역할: 세무·건강보험 전문가 에이전트
> 상태: **초안 (사용자 확인 대기)** — "OK" 확인 후 3단계(데이터 모델·API 스펙 확정)로 진행
> 목적: 백엔드 개발자가 이 문서의 표와 수식을 **그대로 코드(`rules.py`, `tax.py`, `health_insurance.py`)로 옮길 수 있는 수준**의 명세
> 기준 시점: 2025년 제도 기준 (모든 숫자는 `rules.py` 상수로 분리하여 매년 갱신)

---

## 0) 읽는 방법

- 각 모듈은 **입력값 → 계산식 → 출력값** 순서로 기술합니다.
- `RULE:` 표시가 붙은 숫자는 전부 `rules.py`에 상수로 들어갈 값입니다.
- ⚠️ 표시는 "실제 제도와 다른 단순화 포인트"이며, 9장에 전체 목록을 모았습니다.

## 2) 모듈 1: 금융소득 합산 (`calc_financial_income`)

### 입력값

| 필드 | 한국어 의미 | 비고 |
|------|-------------|------|
| `interest` | 이자소득 (예·적금, 채권 이자 등, 연·세전) | |
| `dividend_domestic` | 국내 상장주식 배당금 (연·세전) | |
| `dividend_foreign` | 해외 상장주식 배당금 (연·세전, 원화 환산) | |
| `etf_domestic_equity_dist` | 국내 **주식형** ETF 분배금 | 매매차익은 비과세이므로 입력받지 않음 |
| `etf_domestic_other_dist_and_gain` | 기타 국내 ETF(해외지수·채권·원자재·레버리지·TR·액티브 등)의 분배금 + **보유기간과세 대상 매매차익** | ⚠️ 사용자가 합산해 한 칸에 입력 |
| `etf_foreign_dist` | 해외 상장 ETF 분배금 | 배당소득으로 처리 |
| `capital_gain_foreign` | 해외 상장 주식/ETF **양도차익** (연) | 금융소득 아님, 별도 트랙 |

### 계산식

```
종합과세 대상 금융소득 합계 (financial_income_total) =
    interest
  + dividend_domestic
  + dividend_foreign
  + etf_domestic_equity_dist
  + etf_domestic_other_dist_and_gain
  + etf_foreign_dist

해외 양도소득 (foreign_capital_gain) = capital_gain_foreign   ← 별도 보관, 합산하지 않음
```

### 출력값

```json
{
  "financial_income_total": 21000000,
  "foreign_capital_gain": 10000000
}
```

## 3) 모듈 2: 금융소득종합과세 판정 (`is_subject_to_aggregate_financial_tax`)

### 계산식

```
subject = (financial_income_total > FIN_TAX_THRESHOLD)        # 2,000만 초과 여부
margin  = FIN_TAX_THRESHOLD - financial_income_total          # 음수면 초과액
```

### 출력값

```json
{ "subject": true, "total": 21000000, "threshold": 20000000, "margin": -1000000 }
```

`margin`은 프론트에서 "기준선까지 N원 여유 / N원 초과" 문구에 사용한다.

## 4) 모듈 3: 연금소득 분류 (`classify_pension`)

### 입력값

| 필드 | 의미 |
|------|------|
| `public_annual` | 공적연금 연간 수령액 (국민·공무원·사학·군인) |
| `private_annual` | 사적연금 연간 수령액 (연금저축 + IRP, 세액공제분·운용수익 재원) |
| `tax_free_annual` | 비과세 연금 연간 수령액 (비과세 요건 충족 연금보험 등) |
| `housing_monthly` | 주택연금 월 수령액 |
| `current_age` | 현재 나이 (사적연금 세율 결정용) |

### 계산식

**(a) 공적연금 → 종합과세 대상 금액 (단순화)**

```
public_pension_taxable = max(0, public_annual × PUBLIC_PENSION_TAXABLE_RATIO
                                 - min(public_annual × 0.4, PENSION_INCOME_DEDUCTION_MAX))
```
⚠️ 실제로는 「과세대상 연금액 산정(2002년 이후 납입분) → 연금소득공제(4단계 구간식)」의 2단계지만,
MVP에서는 "수령액의 80%를 과세대상으로 보고, 그중 40%(상한 900만 원)를 공제"로 단순화한다.

**(b) 사적연금 → 분리과세/종합과세 분기**

```
if private_annual <= PRIVATE_PENSION_SEP_LIMIT:           # 1,500만 이하
    private_pension_tax = private_annual × age_rate        # 3.3~5.5% 저율 분리과세로 종결
    private_to_aggregate = 0
else:                                                      # 1,500만 초과
    # MVP 기본값: 16.5% 분리과세 선택 (대부분 유리). ⚠️ 종합과세 선택 비교는 차후 F1
    private_pension_tax = private_annual × PRIVATE_PENSION_EXCESS_SEP_RATE
    private_to_aggregate = 0

age_rate: 70세 미만 0.055 / 70~79세 0.044 / 80세 이상 0.033
```

**(c) 비과세·주택연금**

```
tax_free_cashflow = tax_free_annual + housing_monthly × 12
# 세금 0, 건보 반영 0. 현금흐름에만 더한다.
```

### 출력값

```json
{
  "public_pension_taxable": 7500000,
  "private_pension_tax": 660000,
  "private_to_aggregate": 0,
  "tax_free_cashflow": 6000000,
  "public_annual": 18000000
}
```

## 5) 모듈 4: 종합소득세 추정 (`calc_income_tax`)

### 입력값

근로·사업·기타소득, 임대소득, 모듈 1·3의 출력.

### 계산식 (순서대로)

**1단계 — 종합과세 대상 소득 합산**

```
임대소득(연) = rent_monthly × 12
사업·임대·기타 = business + other + 임대소득(연)
근로소득금액 = max(0, earned − 근로소득공제(단순식))
   근로소득공제(단순식) ⚠️ = min(earned × 0.25, 12,000,000)
       (실제는 5단계 구간식이지만 은퇴 시뮬레이터 특성상 근로소득 비중이 낮아 단순화)

기본소득금액 = 근로소득금액 + 사업·임대·기타 + public_pension_taxable + private_to_aggregate
```

**2단계 — 금융소득 합산 방식 (2,000만 기준 분리)**

```
if financial_income_total <= 2,000만:
    # 전액 분리과세로 종결. 종합소득세에는 미합산
    금융소득_합산분 = 0
    금융소득_분리과세세액 = financial_income_total × 0.14 × 1.1   # 원천징수로 종결된 세금(표시용)
else:
    금융소득_합산분 = financial_income_total − 2,000만          # 초과분만 누진 합산
    기준금액_세액 = 2,000만 × 0.14                               # 2,000만까지는 14%
```

**3단계 — 과세표준과 산출세액**

```
과세표준 = max(0, 기본소득금액 + 금융소득_합산분 − 기본공제)
   기본공제 ⚠️ = 1,500,000 (본인) + (배우자 있고 소득 없으면 1,500,000)
산출세액 = 누진세율표 적용(과세표준) + 기준금액_세액(종합과세 대상인 경우)
종합소득세_추정 = 산출세액 × (1 + LOCAL_TAX_RATE)               # 지방세 포함
```
⚠️ 비교과세(원천징수세액 vs 종합과세 산출세액 중 큰 금액), 배당세액공제(Gross-up)는 생략한다.
생략 방향은 대체로 **세금을 약간 보수적으로(많게) 추정**하는 쪽이다.

**4단계 — 해외 양도소득세 (별도)**

```
해외양도세 = max(0, foreign_capital_gain − FOREIGN_CG_DEDUCTION) × FOREIGN_CG_RATE
```

**5단계 — 연간 총 세부담**

```
total_tax = 종합소득세_추정 + private_pension_tax + 해외양도세
          + (분리과세 금융소득의 원천징수세액)    # 이미 떼인 세금도 현금흐름에서는 차감되므로 포함
```

### 출력값

```json
{
  "aggregate_income_tax": 4500000,
  "private_pension_tax": 660000,
  "foreign_capital_gain_tax": 1650000,
  "financial_withholding_tax": 3080000,
  "total_tax": 9890000,
  "tax_base": 32000000
}
```

## 6) 모듈 5: 건강보험료·피부양자 (`calc_health_insurance`)

### 6-1. 건보 부과 기준 소득 (공통 선행 계산)

```
건보_금융소득 = financial_income_total  if financial_income_total > HI_FIN_INCOME_MIN  else 0
   # 1,000만 초과 시 전액 반영, 이하면 0 (실제 제도와 동일한 구조)

건보_부과기준소득(연) = 건보_금융소득
                      + public_annual × HI_PUBLIC_PENSION_RATIO    # 공적연금 50%
                      + business + other + 임대소득(연)
   # 사적연금·비과세연금·주택연금은 반영하지 않음 (현행 제도와 동일)
```

### 6-2. 자격별 계산

**(a) 직장가입자** (`insurance_status = "직장"`)

```
기본보험료(월) = earned/12 × 0.03545                     # 본인부담분
보수외소득 = 건보_부과기준소득(연)                        # 근로소득 제외 합계
추가보험료(월) = max(0, 보수외소득 − HI_EMPLOYEE_EXTRA_THRESHOLD) / 12 × HI_RATE
월건보료 = (기본보험료 + 추가보험료) × (1 + LTC_RATE_ON_HI)   # 장기요양 포함
```

**(b) 지역가입자** (`insurance_status = "지역"`)

```
소득분(월) = 건보_부과기준소득(연) / 12 × HI_RATE
재산분(월) ⚠️ = max(0, real_estate_tax_base − HI_PROPERTY_DEDUCTION)
              × HI_PROPERTY_RATE_SIMPLE / 12
   # 실제는 60등급 재산점수표 × 점수당 208.4원. MVP는 단순 비율로 근사
월건보료 = max(소득분 + 재산분, HI_REGIONAL_MIN_MONTHLY) × (1 + LTC_RATE_ON_HI)
```

**(c) 피부양자** (`insurance_status = "피부양자"`) → 먼저 유지 가능 여부 판정

### 6-3. 피부양자 유지 판정 (`check_dependent`)

```
판정용_합산소득(연) = financial_income_total                    # ⚠️ 1,000만 기준 없이 전액
                    + public_annual                              # 공적연금 100% 반영 (50% 아님!)
                    + business + other + 임대소득(연)

탈락 조건 (하나라도 해당하면 탈락):
  ① 판정용_합산소득 > DEP_INCOME_LIMIT (2,000만)
  ② 사업소득 > 0  (⚠️ 단순화: 사업자등록 여부 무관하게 사업·임대소득이 있으면 탈락 위험으로 처리)
  ③ real_estate_tax_base > DEP_PROPERTY_LIMIT_2 (9억)
  ④ real_estate_tax_base > DEP_PROPERTY_LIMIT_1 (5.4억)
     이고 판정용_합산소득 > DEP_INCOME_LIMIT_MID_PROPERTY (1,000만)

유지되면: 월건보료 = 0
탈락하면: 지역가입자 (b) 공식으로 재계산하고, "피부양자 탈락 → 지역가입 전환" 플래그 반환
```

> **중요한 비대칭 (사용자가 가장 헷갈리는 부분, UX 문구 필수):**
> - **건보료 부과**에는 공적연금의 **50%**만 반영되지만,
> - **피부양자 판정**에는 공적연금이 **100%** 반영된다.
> - 금융소득도 부과 기준은 "1,000만 초과 시 전액", 판정 기준은 "무조건 전액 합산"으로 다르다.

### 출력값

```json
{
  "status_applied": "지역",
  "monthly_premium": 284000,
  "dependent_eligible": false,
  "dependent_fail_reasons": ["합산소득 2,000만 원 초과"],
  "annual_premium": 3408000
}
```

## 7) 모듈 간 데이터 흐름 (백엔드 구현 순서)

```
입력(UserInput)
  → [모듈1] 금융소득 합산 ─┬→ [모듈2] 종합과세 판정
                            ├→ [모듈4] 소득세 (금융소득 합산분 사용)
                            └→ [모듈5] 건보료 (1,000만/전액 규칙 사용)
  → [모듈3] 연금 분류 ──────┬→ [모듈4] 소득세 (공적연금 과세분)
                            └→ [모듈5] 건보료 (공적연금 50%/100% 규칙)
  → [모듈4]+[모듈5] 결과 → [모듈6: 시뮬레이션, 5차 산출물에서 설계] 연령별 반복 실행
```

연령별 시뮬레이션에서는 매 연도마다 모듈 1~5를 다시 실행한다 (은퇴 전후로 근로소득·자격이 바뀌기 때문).
연령 경과에 따른 자격 자동 전환 규칙:
- 은퇴 나이 도달 → 근로소득 0, 직장가입자였다면 **지역 또는 피부양자 판정**으로 전환 (판정은 모듈 6-3 재사용)
- 공적연금은 사용자가 입력한 수령 개시 나이부터 반영 (기본값 65세)

## 8) 검증용 대표 케이스 3종 (백엔드 테스트에 그대로 사용)

| 케이스 | 입력 요지 | 기대 결과 |
|--------|-----------|-----------|
| ① 안전권 파이어족 | 금융소득 1,800만, 공적연금 0, 재산과표 3억, 피부양자 | 종합과세 **No**, 피부양자 **유지**(소득 2,000만 이하), 건보료 0 |
| ② 경계선 케이스 | 금융소득 2,100만, 공적연금 1,200만, 지역가입 | 종합과세 **Yes**(100만 초과), 건보 부과소득 = 2,100만(전액) + 600만(연금 50%) |
| ③ 피부양자 탈락 | 금융소득 900만, 공적연금 1,300만, 피부양자 | 종합과세 No, 건보_금융소득 0(1,000만 이하)이지만 판정소득 2,200만 → **탈락**, 지역 전환 보험료 산출 |

케이스 ③이 이 시뮬레이터의 존재 이유를 가장 잘 보여준다: **세금은 안 늘었는데 건보료가 새로 생기는** 상황.

## 9) 단순화 포인트 전체 목록 (면책·고지 문구의 근거)

| # | 항목 | 실제 제도 | 본 시뮬레이터 | 오차 방향 |
|---|------|-----------|---------------|-----------|
| S1 | 금융소득 비교과세 | 종합과세 vs 원천징수 중 큰 세액 | 생략 (초과분 누진 + 2,000만×14%) | 거의 동일~약간 과대 |
| S2 | 배당세액공제(Gross-up) | 국내 배당 11% 가산 후 공제 | 생략 | 세금 약간 과대 |
| S3 | 연금소득공제 | 4단계 구간식 | 수령액 80% 과세 가정 + 40%(상한 900만) 공제 | 케이스별 ±수십만 원 |
| S4 | 사적연금 초과 시 선택 | 종합과세 vs 16.5% 분리과세 선택 | 16.5% 분리과세 고정 | 저소득자는 과대 가능 |
| S5 | 근로소득공제 | 5단계 구간식 | 25% (상한 1,200만) | 고소득 구간 과소공제 → 세금 과대 |
| S6 | 인적공제·세액공제 | 자녀·의료비·기부금 등 다수 | 본인+배우자 기본공제만 | 세금 과대 |
| S7 | 지역건보 재산분 | 60등급 점수표 × 208.4원/점 | 과표 1억 공제 후 0.24%/연 단순 부과율 | ±20% 수준 오차 가능 |
| S8 | 임대소득 과세 | 2,000만 이하 분리과세 선택, 필요경비 50~60% | 전액 종합과세, 필요경비 미반영 | 세금 과대 |
| S9 | 피부양자 사업소득 | 사업자등록 시 1원, 미등록 500만 초과 탈락 | 사업·임대소득 있으면 탈락 위험 처리 | 보수적(탈락 과대) |
| S10 | ETF 보유기간과세 | Min(매매차익, 과표증분) | 사용자 입력값 그대로 | 입력 정확도 의존 |
| S11 | 건보 정산·등급경계 | 11월 정산, 점수 경계 효과 | 즉시 반영 | 시점 차이 |
| S12 | 물가·제도 변화 | 매년 요율·기준 변경 | 현재 기준 고정 | 장기 시뮬레이션일수록 오차 누적 |

> 전반적으로 **"세금·건보료를 실제보다 약간 많게(보수적으로) 추정"**하는 방향으로 단순화했다.
> 은퇴 계획 도구에서는 낙관적 오차보다 보수적 오차가 안전하기 때문이다. 이 방향성 자체를 UX 문구에 명시한다.
