# [4차 산출물] 프론트엔드 — 화면 설계 + 구현 안내

> 작성 역할: 프론트엔드 개발자 에이전트 (+ UX·카피 문구 반영)
> 상태: **구현 완료** — `frontend/` 폴더, 빌드 도구 없이 동작하는 단일 페이지
> 근거 문서: `docs/03_데이터모델_API스펙.md` (입출력 JSON 스펙)

---

## 1) 요약

- **단일 HTML 페이지**(`index.html`) + 순수 JavaScript(`app.js`) + CSS(`style.css`). npm·빌드 불필요.
- 그래프는 **Chart.js**를 CDN 스크립트 한 줄로 사용.
- FastAPI 서버가 `/`에서 이 페이지를 직접 서빙하므로, `uvicorn backend.main:app` 실행 후
  `http://localhost:8000` 만 열면 된다.
- 입력값은 서버에 저장되지 않으며, 새로고침하면 사라진다 (개인정보 비저장 원칙).

## 2) 화면 흐름 (6단계 마법사)

```
[면책 배너 - 항상 상단 고정]
Step 1 기본 정보 → Step 2 소득 → Step 3 금융소득 → Step 4 연금
→ Step 5 재산/임대 → [계산하기] → Step 6 결과
```

- 상단에 1~6 단계 진행 표시. 이전/다음 버튼으로 이동, 클릭으로 지난 단계 복귀 가능.
- 모든 금액 입력칸은 **원 단위**이며, 입력 즉시 아래에 "= 2,100만 원" 식으로 읽기 쉬운 단위 표시.
- 금액 필드는 비워두면 0으로 처리 (필수 입력은 나이·건보 자격뿐).

## 3) 결과 화면 (Step 6) 구성

| 영역 | 내용 |
|------|------|
| 판정 카드 3장 | ① 금융소득종합과세 대상 여부 (여유/초과 금액 포함) ② 예상 연간 세금 (구성 내역) ③ 예상 월 건보료 + 피부양자 유지 여부(탈락 사유) |
| 시나리오 비교 표 | A/B/C별: 금융소득, 종합과세 여부, 은퇴 첫해 건보료·순현금흐름, 자산 소진 나이, 90세 잔액 |
| 그래프 2개 | ① 연령별 순 현금흐름 (시나리오 3개 라인) ② 연령별 금융자산 잔액 |
| 연도별 상세 표 | 시나리오 선택 드롭다운 → 나이/자격/수입/세금/건보료/순현금/잔액 |
| 단순화 고지 | 접이식 "이 계산이 실제와 다를 수 있는 이유" (docs/02 9장 요약) |

## 4) API 연동

- `[계산하기]` 클릭 → 입력 수집 → `POST /api/simulate` (JSON) → 응답 렌더링.
- 422 검증 오류·네트워크 오류 시 빨간 오류 박스로 한국어 안내.

## 5) 파일 구조

> 참고: 버셀(Vercel) 배포를 위해 폴더명이 `frontend/` → `public/`으로 바뀌었다 (버셀의 정적 파일 규칙).

```
public/
├── index.html        # 전체 마크업 (입력 필드·안내 문구 수정은 이 파일)
└── static/
    ├── app.js        # 단계 이동, 금액 표시, API 호출, 결과·그래프 렌더링
    └── style.css     # 색·간격 등 외관 (파란 계열, 카드 UI)
```

비개발자 수정 가이드: **문구를 바꾸고 싶으면 `index.html`**, **숫자(세율 등)는 `backend/rules.py`**,
**색·디자인은 `style.css`**만 보면 된다.
