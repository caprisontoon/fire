# [5차 산출물] 데이터/시뮬레이션 — 연령별 현금흐름·시나리오 설계

> 작성 역할: 데이터/시뮬레이션 설계 에이전트
> 상태: **구현 완료** — `backend/simulation.py`에 반영됨. 본 문서는 가정과 수식의 설명서
> 근거: docs/02 7장(모듈 간 흐름), docs/03 3장(출력 구조)

---

## 1) 요약

- **기간:** 현재 나이 ~ 90세, 연 단위. 매 연도마다 세금·건보 계산(모듈 1~5)을 처음부터 다시 실행한다.
- **시나리오 3종:** A(입력 그대로) / B(금융소득 연 2,000만 이하 관리) / C(연 1,000만 이하 관리).
- **자산 변화:** 연 4% 명목 수익률 가정. 생활비를 입력하면 "자산이 몇 살까지 버티는지"(소진 나이)를 계산.

## 2) 연도별 상태 전환 규칙

| 사건 | 시점 | 처리 |
|------|------|------|
| 은퇴 | `retire_age` 도달 | 근로소득 0. 사업·기타·임대·금융소득은 계속 |
| 사적·비과세·주택연금 개시 | `retire_age`부터 | ⚠️ 단순화: 은퇴와 동시 개시로 가정 |
| 공적연금 개시 | `public_pension_start_age`부터 (기본 65세) | 개시 전에는 소득·건보 모두 미반영 |
| 건보 자격 전환 | 은퇴 후 매년 | 피부양자 조건 충족 → 피부양자(보험료 0), 미충족 → 지역가입자 |
| 사적연금 세율 변화 | 70세·80세 도달 | 5.5% → 4.4% → 3.3% 자동 적용 |

은퇴 후 자격을 매년 다시 판정하므로, **연금 개시 등으로 소득이 늘어나는 해에 피부양자가 박탈되는 시점**이 표에 그대로 드러난다.

## 3) 자산 변화식 (이중계산 방지가 핵심)

사용자가 입력한 금융소득(이자·배당·분배금)과 해외 양도차익은 **금융자산에서 나오는 수익의 일부를 현금으로 꺼낸 것**으로 본다. 따라서:

```
다음해 자산 = 자산 × (1 + 4%)              ← 자산 전체의 수익
            − (금융소득 + 해외 양도차익)     ← 현금으로 빠져나간 부분 (수입 쪽에 이미 계상)
            + (순 현금흐름 − 연간 생활비)    ← 쓰고 남으면 재투자, 모자라면 자산 인출
```

- 입력한 금융소득이 자산의 4%보다 크면 사실상 원금을 헐어 쓰는 셈이 되어 자산이 줄어든다 — 4% 룰의 논리가 자연스럽게 내장된다.
- **자산 소진 나이** = 잔액이 처음 0 미만이 되는 나이. 90세까지 버티면 "유지"로 표시.
- 생활비를 0으로 두면 소진 계산은 의미가 없으므로 생략한다.

## 4) 시나리오 B·C의 구현 방식

- 금융소득 6개 입력 필드를 **같은 비율로 축소**해 합계가 한도(B: 2,000만 / C: 1,000만)와 같아지게 만든다.
- 해외 양도차익은 금융소득이 아니므로 줄이지 않는다 (해외 ETF 매매 중심 전략과 일관).
- 이미 한도 이하라면 그대로 둔다 (B·C가 A와 같아짐).
- ⚠️ 해석 주의: "금융소득을 줄인다"는 것은 현실에서는 배당주 비중 축소, 연금·ISA 계좌로 이동,
  매도 시점 분산 등을 의미한다. 시뮬레이터는 줄어든 **결과**만 반영하며, 줄이는 **방법**은 다루지 않는다.
  또한 줄어든 금융소득만큼 현금 수입도 줄어든 것으로 처리한다 (자산에 남아 복리로 굴러간다).

## 5) 고정 가정값과 한계

| 가정 | 값 | 한계 |
|------|-----|------|
| 자산 수익률 | 연 4% (명목, `rules.ASSET_RETURN_RATE`) | 변동성 없음. 실제 시장의 순서위험(초반 폭락) 미반영 |
| 물가상승률 | 0 (미반영) | 모든 금액은 현재 가치 기준으로 읽어야 함 |
| 연금 인상률 | 0 | 국민연금의 물가연동 미반영 (물가 0 가정과 상쇄되는 방향) |
| 소득 구성 | 매년 동일 | 임대 종료, 배당 변동 등 미반영 |
| 제도 | 2025년 기준 고정 | 세율·건보 기준 개정 미반영 |

차후 확장(F4)에서 수익률·물가를 사용자 입력으로 열 수 있도록 `rules.py`에 상수로 분리해 두었다.
